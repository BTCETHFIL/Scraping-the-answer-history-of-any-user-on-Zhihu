# 知乎内容归档工具

> 自动保存任意知乎用户的回答历史为本地文档，每篇回答同时生成高清截图和可搜索文字。
> 提供图形界面和命令行两种方式，适合个人阅读备份和资料整理。

---

## 🆕 核心设计：滚屏与输出分离

GUI 将「收集链接」与「生成 MD」拆为两个独立步骤，一次滚屏可反复用不同关键词生成多份 MD：

```
添加用户 → 开始滚屏（全量收集链接到缓存）
                ↓
         选关键词（可选）
                ↓
         从缓存生成 MD（可多次、不同关键词）
```

**三个操作互斥**：「开始滚屏」「从缓存生成 MD」「手动粘贴链接」同一时间只能进行一个。

---

## 能做什么

| 功能 | 说明 |
|------|------|
| **滚屏收集链接** | 自动滚动加载用户全部回答链接，存入本地缓存（不生成 MD） |
| **缓存生成 MD** | 从已缓存的链接按需生成 Markdown，不选关键词 = 全量输出 |
| **截图+文字双输出** | 每篇回答同时保存整页截图（PNG）+ 可搜索 Markdown 文字 |
| **图片内嵌** | 回答中的图片以 Base64 嵌入文档，单文件自包含 |
| **增量更新** | 再次运行只获取新回答，已保存的不重复处理 |
| **多用户批量** | 一次配置多位用户，依次自动完成 |
| **三种登录方式** | 扫码登录 / 手动 Cookie / 复用上次登录态 |
| **断点续传** | 进度自动记录，中断后可无缝继续 |
| **智能缓存** | 30 分钟内重复运行跳过网络请求，减少访问次数 |
| **快速测试** | 仅滚屏 3 条链接，调试验证配置 |
| **关键词过滤** | 多关键词（逗号/顿号/空格/分号分隔），不区分大小写，标题/内容含任一即保存 |
| **关键词分组管理** | 保存为命名分组（如"蔚来相关"）、一键应用、从文件批量导入、最近使用记录 |
| **用户分组管理** | 保存为命名分组（如"蔚来黑子"）、一键应用/替换/合并、从文件批量导入、最近使用记录 |
| **已抓取用户列表** | 自动生成统计报告，含每个用户基本信息、抓取次数、累计回答数、本地文件校验 |
| **目录自动合并** | 同一用户昵称变化时自动合并旧目录到新目录，避免重复文件夹 |
| **昵称自动清洗** | 抓取时自动过滤混入昵称的个人简介/签名，保持目录名干净 |
| **大文件分拆** | 超过 10MB 的文档可自动按标题分割 |
| **法务证据模式** | 保存原始 HTML + 生成文件完整性校验报告 |

## 安装和启动

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 安装浏览器驱动（首次运行自动安装）
playwright install chromium

# 3. 启动图形界面（推荐，直接双击运行）
python 双击运行.py

# 4. 或者命令行模式
python -m src.main <用户ID>
```

## 图形界面说明

启动后看到主窗口，分为配置面板（左侧）和运行日志（右侧）。左侧分为三个功能区：

### 📋 一、目标用户管理
| 操作 | 说明 |
|------|------|
| ➕ 添加到列表 | 输入用户主页 URL 或用户 ID，快速添加 |
| 全选 / 全不选 | 批量选择要处理的用户 |
| ✏ 编辑昵称 | 修改用户的备注名称 |
| 🏠 访问主页 | 在浏览器中打开选中用户的知乎主页 |
| 🔒 生成HTML | 将已抓取的回答从缓存生成法务证据 .html 文件，无需重新爬取 |
| 🗑 删除 | 从列表中移除用户 |
| 🧹 清除缓存 | 清除选中用户的滚屏链接缓存，下次重爬全量 |
| 🔄 刷新列表 | 刷新用户列表，显示最新爬取历史 |
| 👥 用户分组 | 保存/应用/管理用户分组，支持从 .txt 批量导入、最近使用恢复 |

### 🔑 登录设置（三选一）

| 方式 | 适用场景 | 说明 |
|------|----------|------|
| 扫码登录 | 桌面环境 | 点击「开始」后弹出浏览器扫码 |
| 手动 Cookie | 无桌面/服务器 | 从浏览器开发者工具复制 Cookie 填入 |
| 复用登录态 | 二次使用 | 上次登录成功后自动保存，无需重新登录 |

### ⚙️ 二、爬取（滚屏）设置 — 全量收集链接，不生成 MD

| 设置项 | 说明 |
|------|------|
| **最多滚屏** | 限制每人滚屏收集的回答条数（0=不限） |
| **输出目录** | 自定义保存路径（默认 `output/`） |
| **Chrome 路径** | 指定已安装的 Chrome 浏览器，无需额外下载 |
| **测试模式** | 只滚屏前 3 条链接，快速验证 |
| **法务证据模式** | 保存原始 HTML + 完整性报告 |
| **延迟策略** | 滚动间隔（2-5s）和翻页间隔（3-8s）随机化 |
| **缓存有效期** | 默认 30 分钟，设为 0 禁用 |

### 📝 三、输出 MD — 从缓存生成（可选关键词筛选）

| 设置项 | 说明 |
|------|------|
| **关键词** | 输入关键词过滤内容（标题/回答匹配任一即保存，不区分大小写）；**不填 = 全量输出** |
| **📂 分组** | 保存/应用/管理关键词分组，支持从 .txt 文件批量导入 |
| **🕐 最近** | 下拉选择最近使用过的关键词组合 |
| **▶ 从缓存生成 MD** | 对选中的用户，从滚屏缓存读取链接，按关键词筛选后生成 Markdown |
| **🔗 手动粘贴链接** | 直接在文本框粘贴知乎回答链接，逐条抓取保存（最多 100 条） |

### ▶ 操作控制
| 按钮 | 说明 |
|------|------|
| ▶ 开始滚屏 | 启动滚屏任务（只收集链接，不生成 MD） |
| ▶ 继续 | 断点续传：从上次停止位置继续滚屏 |
| ■ 停止 | 安全中止当前任务 |
| ▶ 从缓存生成 MD | 读取已缓存的链接，生成 Markdown 文件 |
| 📝 批量保存为 MD | 粘贴链接后逐条抓取保存 |
| 🔒 生成HTML | 从缓存补生成法务证据 .html 文件（无需重爬） |
| ✂ 分割大文档 | 将超过 10MB 的文档按标题自动分割 |

> **三者互斥**：「开始滚屏」「从缓存生成 MD」「批量保存为 MD」同一时间只能运行一个。

## 输出结构

```
output/
└── {昵称}_{用户ID}/                 # 用户目录（昵称便于识别，ID 确保唯一）
    ├── cache/                       # 短时缓存（24h 有效）
    │   ├── links.json               #   回答链接列表缓存
    │   └── {answer_id}.json         #   单条回答内容缓存
    ├── INDEX.md                     # 回答索引（按日期排列）
    ├── progress.json                # 进度记录（断点续传用）
    ├── 2024-01-15_如何评价XXX.md    # Markdown 文档（截图引用 + 文字）
    ├── {answer_id}.png              # 整页高清截图
    ├── 2024-01-15_如何评价XXX.html  # 法务模式：原始 HTML 副本
    └── evidence_report.json         # 证据完整性报告（SHA-256）
```

> 昵称变化时自动合并旧目录到新目录，保证同一用户只有一个目录。

每篇文档包含：
- **回答全文**（可搜索、可复制）
- **截图引用**（相对路径，本地查看即可显示）
- **用户影响力数据**（赞同数、评论数、收藏数等）
- **发布时间**和**编辑时间**

## 辅助功能

### 📊 已抓取用户列表
每次爬取后自动生成 `已抓取用户列表.md`，包含：
- **总览统计**：总用户数、已抓取回答数、现存 MD 文件数
- **逐用户明细**：头像、昵称、用户 ID、最近抓取时间、回答数（日志 vs 本地文件交叉校验）
- **待抓取列表**：已添加但尚未爬取的用户

### 📝 手动链接保存
无需预先配置用户 ID，直接在文本框粘贴知乎回答链接（每行一条，最多 100 条），点击「📝 批量保存为 MD」即可逐条获取并生成文档。超 100 条自动分批处理。

支持格式：
- `https://www.zhihu.com/question/XXXXXX/answer/YYYYYY`
- `https://zhuanlan.zhihu.com/p/ZZZZZZ`
- App 复制格式（文字描述 + 链接混合粘贴）
- 从 .txt/.csv 文件批量导入链接

### ✂ 大文件分拆
点击「✂ 分割大文档」按钮，选择目录后自动扫描超过 10MB 的 Markdown 文件，按二级标题（## ）智能分割为多个小文件，方便阅读器打开。

### 🔒 法务证据模式
开启后额外保存：
- 回答页的**原始 HTML 副本**
- **证据完整性报告**（含各文件的 SHA-256 哈希值，可验证未被篡改）
- 用户**影响力数据汇总**

> 💡 如果先以默认模式抓取，后需补生成法务 HTML：选中用户后点击「🔒 生成HTML」按钮，从缓存中生成缺失的 .html 文件，无需重新爬取。

## 命令行模式

适合服务器或自动化场景：

```bash
# 单用户
python -m src.main zhang-jia-wei

# 多用户批量
python -m src.main zhang-jia-wei liu-bo-wen-27

# 指定配置文件
python -m src.main --config my_config.json

# 手动链接模式
python -m src.main --url https://www.zhihu.com/question/xxx/answer/yyy
```

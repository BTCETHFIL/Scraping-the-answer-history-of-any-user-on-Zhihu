# 知乎内容归档工具

> 自动保存任意知乎用户的回答历史为本地文档，每篇回答同时生成高清截图和可搜索文字。
> 提供图形界面和命令行两种方式，适合个人阅读备份和资料整理。

---

## 能做什么

| 功能 | 说明 |
|------|------|
| **全量回答保存** | 自动加载用户全部回答，无需手动翻页 |
| **截图+文字双输出** | 每篇回答同时保存整页截图（PNG）+ 可搜索 Markdown 文字 |
| **图片内嵌** | 回答中的图片以 Base64 嵌入文档，单文件自包含 |
| **增量更新** | 再次运行只获取新回答，已保存的不重复处理 |
| **多用户批量** | 一次配置多位用户，依次自动完成 |
| **三种登录方式** | 扫码登录 / 手动 Cookie / 复用上次登录态 |
| **断点续传** | 进度自动记录，中断后可无缝继续 |
| **智能缓存** | 30 分钟内重复运行跳过网络请求，减少访问次数 |
| **快速测试** | 仅获取 3 条，调试验证配置 |
| **关键词过滤** | 多关键词（逗号/顿号/空格/分号分隔），不区分大小写，标题/内容含任一即保存 |
| **关键词分组管理** | 保存为命名分组（如"蔚来相关"）、一键应用、从文件批量导入、最近使用记录 |
| **已抓取用户列表** | 自动生成统计报告，含每个用户基本信息、抓取次数、累计回答数 |
| **大文件分拆** | 超过 10MB 的文档可自动按标题分割 |
| **法务证据模式** | 保存原始 HTML + 生成文件完整性校验报告 |

## 安装和启动

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 安装浏览器驱动（首次运行自动安装）
playwright install chromium

# 3. 启动图形界面（推荐）
python gui.py

# 4. 或者命令行模式
python main.py <用户ID>
```

## 图形界面说明

启动后看到主窗口，分为配置面板（左侧）和运行日志（右侧）：

### 📋 目标用户管理
| 操作 | 说明 |
|------|------|
| ➕ 添加到列表 | 输入用户主页 URL 或用户 ID，快速添加 |
| 全选 / 全不选 | 批量选择要处理的用户 |
| ✏ 编辑昵称 | 修改用户的备注名称 |
| 🗑 删除 | 从列表中移除用户 |

### 🔑 登录设置（三选一）

| 方式 | 适用场景 | 说明 |
|------|----------|------|
| 扫码登录 | 桌面环境 | 点击「开始」后弹出浏览器扫码 |
| 手动 Cookie | 无桌面/服务器 | 从浏览器开发者工具复制 Cookie 填入 |
| 复用登录态 | 二次使用 | 上次登录成功后自动保存，无需重新登录 |

### ⚙️ 参数配置
| 设置项 | 说明 |
|------|------|
| **最多获取** | 限制每人获取的回答条数（0=不限） |
| **输出目录** | 自定义保存路径（默认 `output/`） |
| **Chrome 路径** | 指定已安装的 Chrome 浏览器，无需额外下载 |
| **无头模式** | 后台运行不显示浏览器窗口 |
| **测试模式** | 只获取前 3 条回答，快速验证 |
| **关键词** | 输入关键词过滤内容（标题/回答匹配任一即保存，不区分大小写） |
| **📂 分组** | 保存/应用/管理关键词分组，支持从 .txt 文件批量导入 |
| **🕐 最近** | 下拉选择最近使用过的关键词组合 |
| **法务证据模式** | 保存原始 HTML + 完整性报告 |
| **缓存有效期** | 默认 30 分钟，设为 0 禁用 |
| **强制忽略缓存** | 跳过所有缓存从头处理 |

### ▶ 操作控制
| 按钮 | 说明 |
|------|------|
| ▶ 开始采集 | 启动任务 |
| ■ 停止 | 安全中止 |
| 📝 批量保存链接 | 粘贴知乎回答链接，逐条获取保存（最多 20 条） |
| ✂ 分割大文档 | 将超过 10MB 的文档按标题自动分割 |

## 输出结构

```
output/
└── zhang-jia-wei/                   # 用户标识目录
    ├── INDEX.md                     # 回答索引（按日期排列）
    ├── progress.json                # 进度记录（断点续传用）
    ├── 2024-01-15_如何评价XXX.md    # Markdown 文档（截图引用 + 文字）
    ├── 2024-01-15_如何评价XXX.png   # 整页高清截图
    └── ...
```

每篇文档包含：
- **回答全文**（可搜索、可复制）
- **截图引用**（相对路径，本地查看即可显示）
- **用户影响力数据**（赞同数、评论数、收藏数等）
- **发布时间**和**编辑时间**

## 辅助功能

### 📝 手动链接保存
无需预先配置用户 ID，直接在文本框粘贴知乎回答链接（每行一条，最多 20 条），点击「批量保存」即可逐条获取并生成文档。

支持格式：
- `https://www.zhihu.com/question/XXXXXX/answer/YYYYYY`
- `https://zhuanlan.zhihu.com/p/ZZZZZZ`

### ✂ 大文件分拆
点击「✂ 分割大文档」按钮，选择目录后自动扫描超过 10MB 的 Markdown 文件，按二级标题（## ）智能分割为多个小文件，方便阅读器打开。

### 🔒 法务证据模式
开启后额外保存：
- 回答页的**原始 HTML 副本**
- **证据完整性报告**（含各文件的 SHA-256 哈希值，可验证未被篡改）
- 用户**影响力数据汇总**

## 命令行模式

适合服务器或自动化场景：

```bash

# 多用户批量

python main.py --config my_config.json

python main.py --url https://www.zhihu.com/question/xxx/answer/yyy
```

## 注意事项

1. **仅供个人备份使用**，请尊重原作者版权，勿用于商业用途
2. **访问频率控制**：工具内置随机延迟策略（滚动间隔 2-5 秒，翻页间隔 3-8 秒）
3. **需要登录**：知乎部分内容需要登录后才能查看，推荐使用扫码登录
4. **Chrome 浏览器**：可使用系统已安装的 Chrome（在配置中指定路径），也可使用 Playwright 自带的 Chromium
5. **网络环境**：需要能够正常访问知乎网站
